This is the README file for exam project number 22.
This project contains the implementation of an open 4-point recursive integrator with nodes 
{1/5, 2/5, 3/5, 4/5}. The file 06_erf_acc.dat is taken as a result from the 
Quad homework, and used for comparison.

We start with a bit of numerical integration mathematics
(Chat GPT Instant 5.3 assisted with the reasoning behind the theory).

Let h=b-a be the height then we have four open nodes
x1=a+1/5h, x2=a+2/5h, x3=a+3/5h, and x4=a+4/5h.
Since the nodes are symmetric around 1/2 and we choose to work on the normalized 
[0, 1] then the Riemannian integral becomes
Af(1/5) + Bf(2/5) + Bf(3/5) + Af(4/5). [eq. 1]
By demanding that when f(x)=1 the quadrature gives us 
2A + 2B = 1 which implies A + B = 1/2.
For f(x)=x^2 we must have that the integral on this interval is equal to 1/3. 
We use equation 1. and find
17A + 13B = 25/3 which implies A = 11/24, B = 1/24.
Therefore do we have the weights w1 = w4 = 11/24, and w2 = w3 = 1/24.

The implementation (integ_open.hpp):

This was utilized by first implementing the integrate function, which calculates 
first h and then f1, f2, f3, f4. The four f-values are the fed into the adapt 
function where we split the interval from [a, b] into two halves with midpoint
m = (a+b)/2. A first estimate of Q is obtained for the f-values. 
The crucial step is to define a left and right child where then four new f-values 
are calculated for both left and right respectively. 
For left and right the Q_l, and Q_r is obtained using the updated h_l, and h_r
respectively.
The sum is taken of both Q-values and the error from the estimate is found, and
if the error is under a certain tolorance are we done. Otherwise do the right and
left f-values get fed into the adapt function again (recursion), and the process 
starts over.
Also included in the header file are the Clenshaw-Curtis integrator for singular 
integrals and the error function. 

Results:

Firstly, the routine was applied to erf(z), and yielded a promising plot 
(erf_plot_open.pdf). All the other plots are comparisons with the old 4-point
integrator with nodes {1/6,2/6,4/6,5/6} (homework 06 Quad, file: quadinteg.hpp).

With the plot, erf_plot_open_acc.pdf, we compared the error of erf(1), 
where we see that the new 4-point integrator yielded much better results as 
in the extremes of acc~1e-8 lies two magnitued lower in error, while also having 
a significantly lower error at a~1e-3 to 1e-1.
In plot, plot_erf_calls.pdf (error vs number of calls), we see that the new 
implementation yielded less error for a significantly lower number of calls.
The old- and new integrator were then set to find the integrals of sqrt(x), and 
sqrt(1-x^2), both on the interval of [0, 1]. When plotted we see nearly identical
behavior; for a low number of calls, around <50, the old integrator has slightly
lower error, and when the number of calls surpasses 100, the new integrator has
1-2 less magnitudes in error. 

For more singular integrals as 1/sqrt(x) and ln(x)/sqrt(x) the Clenshaw-Curtis 
routine was used. Here the new integrator yields results of much less error for 
1/sqrt(x) (plot_invsqrt_calls.pdf). The new integrator has shown limitations 
when presented with ln(x)/sqrt(x), on the ground of the singularity at x=0, 
which we can see in plot_ln_sqrt_calls.pdf. In this case the old integrator is
superior in that it does not exceed the recursion depth.

To sum up the the new implemented and recursive 4-point integrator with nodes 
{1/5,2/5,3/5,4/5} is more efficient (lower number of calls) and of less error, 
but the implementation is limited when presented with a singular type integral 
as ln(x)/sqrt(x), since when x-->0 then ln(x) approaches negative infinity much 
faster than sqrt(x) approaches 0.
